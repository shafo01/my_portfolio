My Git Workflow

I work in a small team with awesome team-mates. In our team, there are three developers, myself included. To collaborate we use github. When starting to use git, I did face some challenges.

Firstly, I did not fully grasp the concepts and flow of git. Though I still used to make changes, create features and merge my changes with the master branch on the repo, I always had an uncertain and hesistant feeling.I am sort of a visual person and, when it comes to processes, I like to have a clear picture in my head. Hence, I decided to learn more about git.

I followed some of the lessons from two of Kevin Skoglund courses on LinkedIn Learning. I enjoyed watching those lessons and I definitely recommend them to anyone. He clearly described the process and gave clear instructions. The two courses are:

  • Git Essential Training: The Basics
  • Git Branches, Merges, and remotes

After watching these videos, I was able to more clearly visualize what was happening whilst I was pushing to the git remote. Here are commands that I often use to update my latest changes to the master on our github repo.

Git Clone

If you want to create a copy of the remote repository, instead of downloading the remote repository, you can simply clone.

  • Navigate (cd) into a directory that will become the parent directory.
  • Go to your github repository. Click on the 'Clone or download' button. There you will see the https web url. Copy it.
  • Go back to the terminal and enter the following command:
    git clone https_url_of_github_repository*
    * This will be the actual https url from your git hub

Status of local repository

  • Checking the status of the local repository is a great way to check all the changes that we made into our local branches.
    git status

Save changes on local repository

  • Add changes:
    git add .
    or
    git add file_name*
    * The name(including the extension) of the file you want to add. If you wish to add more than one file then use a space between the file names to separate them.

  • Commit / save the changes:
    git commit -m "headline
    description (optional)"

Compare changes with working directory and origin

  • git diff master origin/master

Create new branch in local working directory

  • git checkout -b name_of_branch

Delete a branch in local working directory

  • git branch -d name_of_branch

See all branches in the local working directory

  • git branch

Go to a specific branch

  • git checkout name_of_branch

Push changes to the remote repository on Github

  • git push -u origin name_of_branch

On Remote Repository, merge branch with master

For the remote repository on Github, when I have to merge a branch that is already on Github, I simply use the Github for merging the branch into master.

Delete a remote branch from terminal

  • git push origin --delete name_of_branch

Update local branch with Remote Repository on Github

It is recommended to always update your local repository before you start work or attempt to make any push.
Use either a combination of Git fetch and merge or use git pull. Because I usually like to check the difference in between I usually like to use git fetch and merge.

  • Git fetch and merge
    git fetch
    git merge
  • Git pull
    git pull
